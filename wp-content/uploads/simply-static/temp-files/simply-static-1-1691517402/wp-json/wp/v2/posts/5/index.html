PEP 484<\/strong><\/a> previously introduced TypeVar<\/code><\/a>, enabling creation of generics parameterised with a single type. PEP 646<\/strong><\/a> adds TypeVarTuple<\/code><\/a>, enabling parameterisation with an arbitrary<\/em> number of types. In other words, a TypeVarTuple<\/code><\/a> is a variadic<\/em> type variable, enabling variadic<\/em> generics.<\/p>\n\n\n\n

This enables a wide variety of use cases. In particular, it allows the type of array-like structures in numerical computing libraries such as NumPy and TensorFlow to be parameterised with the array shape<\/em>. Static type checkers will now be able to catch shape-related bugs in code that uses these libraries.<\/p>\n\n\n\n

See PEP 646<\/strong><\/a> for more details.<\/p>\n\n\n\n
